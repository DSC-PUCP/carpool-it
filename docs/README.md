# Docs

Esta carpeta centraliza la documentacion tecnica del proyecto para facilitar onboarding y colaboracion.

## Contenido

- `architecture.md`: arquitectura del sistema (frontend, SSR, Cloudflare, Supabase).
- `folder-structure.md`: estructura de carpetas y responsabilidad de cada modulo.
- `data-model.md`: modelo de datos actual en Supabase (tablas, relaciones y reglas).
- `supabase-local.md`: setup local de Supabase y configuracion de Google OAuth.

## Uso recomendado para nuevos devs

1. Leer `architecture.md` para entender el flujo general.
2. Revisar `folder-structure.md` para saber donde implementar cada cambio.
3. Revisar `data-model.md` antes de tocar repositorios, servicios o consultas SQL.
4. Seguir `supabase-local.md` para levantar entorno local de backend y auth.
